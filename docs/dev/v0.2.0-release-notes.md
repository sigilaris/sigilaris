# Sigilaris v0.2.0 Release Notes

## Release Summary
- Release date: 2026-04-16
- Artifact version: `0.2.0`
- Scope: publish the node/runtime baseline that landed after `v0.1.1`, including the cross-platform `node-common` surface, JVM HotStuff runtime baseline, and the refreshed public site/docs structure.

## Highlights
- Added the new published cross-platform artifact `sigilaris-node-common` for transport-neutral gossip/session/bootstrap contracts and tx anti-entropy runtime logic.
- Rebased `sigilaris-node-jvm` on top of `sigilaris-node-common`, with JVM-only runtime assembly, Armeria transport adapters, config loading, and storage integrations kept in the JVM module.
- Shipped the current JVM HotStuff baseline:
  - canonical block model and `BlockStore`
  - proposal / vote / QC artifact flow
  - pacemaker timeout / new-view progression
  - snapshot sync, forward catch-up, and background historical backfill
  - static peer principal binding, HMAC transport auth, and session-bound bootstrap capability checks
- Added conflict-aware application scheduling primitives and runtime support around `StateRef`, `ConflictFootprint`, batch planning, and receipt projection cleanup.
- Realigned the Typelevel site and bilingual docs to the current module structure, including new `node-common` and `node-jvm` narrative sections.

## Public Contract Notes
- Published coordinates now include:
  - `org.sigilaris:sigilaris-core_3`
  - `org.sigilaris:sigilaris-core_sjs1_3`
  - `org.sigilaris:sigilaris-node-common_3`
  - `org.sigilaris:sigilaris-node-common_sjs1_3`
  - `org.sigilaris:sigilaris-node-jvm_3`
- The transport-neutral gossip/session model now lives under `org.sigilaris.node.gossip` and `org.sigilaris.node.gossip.tx`.
- JVM-only runtime/bootstrap/transport/storage implementations remain under `org.sigilaris.node.jvm.*`.
- Static topology configuration is read from `sigilaris.node.gossip.peers`; the shipped HotStuff baseline is read from `sigilaris.node.consensus.hotstuff`.

## Upgrade Notes
- Update build definitions to `0.2.0`.
- Add `sigilaris-node-common` when you need the shared node contracts directly on JVM or Scala.js; `sigilaris-node-jvm` already depends on it transitively.
- If you consume internal/runtime packages, review imports for the cross-runtime split between `org.sigilaris.node.gossip.*` and `org.sigilaris.node.jvm.*`.
- If you adopt the JVM runtime baseline, provide the static peer inventory, transport-auth peer secrets, validator inventory, key-holder mapping, and signer configuration expected by the current config loaders.
- If you integrate the application execution pipeline directly, review the conflict-scheduling and receipt-projection changes before upgrading.

## Verification Snapshot
- Core JVM/JS suites passed for the current application scheduling, datatype/codec/failure regressions, and accounts/group integration coverage.
- `node-common` JVM/JS suites passed for gossip model, session engine, and tx anti-entropy runtime coverage.
- `node-jvm` suites passed for HotStuff validation/runtime/bootstrap/materialization/storage/transport coverage, including the static multi-node launch smoke harness.
- The Typelevel site build completed with API docs mapped into the generated site output.
