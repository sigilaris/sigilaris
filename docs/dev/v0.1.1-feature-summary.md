# v0.1.1 Feature Snapshot

## Overview
- Scope: capture the features already landed and slated for the v0.1.1 release.

## Items Completed
- TxEnvelope relocation: moved the transaction envelope implementation into the shared core module so downstream components can reuse a single canonical representation.
- NetworkId definition: introduced a unified network identifier type with strong typing and validation hooks to eliminate ad-hoc string handling across modules.

## In Progress
- Codec additions: expand the core codec suite with Boolean and single-byte primitives to replace custom serializers and unify encoding logic.

## Next Steps
- Validate integration points that still reference legacy TxEnvelope paths.
- Ensure NetworkId usage is documented in public API guides before release freeze.
- Prototype a neotype-style companion mix-in that auto-derives byte codecs, JSON codecs, and Eq for opaque wrappers.
- Design a scodec-inspired DiscriminatorCodec to streamline sum-type encoding selection.
- Introduce a derived helper on `ByteCodec` so opaque products can summon Mirror-based instances without boilerplate.
- Explore representing schema definitions as case classes of `(K_i, V_i)` pairs to simplify tuple conversion.
- Add a compile-time guard via `TxHandler.ensureAllHandlers[Tx]` (using Scala 3 `Mirror`) so every transaction subtype ships with a handler instance.

```scala
import scala.deriving.Mirror

sealed trait Tx
final case class CreateNamedAccount(...) extends Tx
final case class UpdateAccount(...) extends Tx

trait TxHandler[T <: Tx]:
	def handle(tx: T): Unit

object TxHandler:
	inline def ensureAllHandlers[T <: Tx](using m: Mirror.Sum.Of[T]): Unit =
		inline def loop[Elems <: Tuple]: Unit = inline erasedValue[Elems] match
			case _: (h *: t) =>
				summonInline[TxHandler[h & T]]
				loop[t]
			case _: EmptyTuple =>
				()

		loop[m.MirroredElemTypes]

// Fails to compile if any Tx subtype lacks an implicit TxHandler
TxHandler.ensureAllHandlers[Tx]
```
