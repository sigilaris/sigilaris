# v0.1.1 Feature Snapshot

## Overview
- Scope: capture the features already landed and slated for the v0.1.1 release.

## Items Completed
- TxEnvelope relocation: moved the transaction envelope implementation into the shared core module so downstream components can reuse a single canonical representation.
- NetworkId definition: introduced a unified network identifier type with strong typing and validation hooks to eliminate ad-hoc string handling across modules.
- Primitive codec completion: added the missing canonical `Boolean` byte codec (`false -> 0x00`, `true -> 0x01`) and locked the regression tests/docs around that single-byte contract.
- JSON discriminator contract consolidation: extracted the shared sealed trait/enum subtype-label projection used by wrapped-by-type-key derivation, keeping canonical Mirror labels as the default public contract.
- Opaque derivation foundations: added reusable opaque companion mix-ins for representation-backed wrappers and applied them to `GroupId` and `NetworkId`, while leaving special-layout codecs on manual implementations.

## In Progress
- Current focus, Phase 3: wire compile-time registry coverage into `TxRegistry` and introduce stable failure metadata without breaking existing `msg`-based APIs.

## Phase 0 Decisions Locked
- JSON coproducts keep `wrapped-by-type-key` as the only public representation for `v0.1.1`.
- Canonical subtype labels come directly from Scala 3 Mirror labels for both sealed traits and enums; renames are opt-in via explicit `TypeNameStrategy.Custom` mappings only.
- Opaque wrapper ergonomics will use a single representation-driven companion trait family that forwards `Eq`, `ByteCodec`, `JsonCodec`, and optional `JsonKeyCodec` from the representation type.
- `FailureCode` remains transport-neutral and core-owned; `ErrorKey` is the normalized external key projected by formatters/adapters.
- Reusable codec law helpers will stay as internal test support in `modules/core/shared/src/test/scala` for `v0.1.1`; no new publishable testkit module will be added in this milestone.

## Recommended Additions For v0.1.1
- Sum-type and enum contract consolidation: core JSON derivation already supports wrapped-by-type-key sums, so the v0.1.1 opportunity is to extract the discriminator/label strategy into a reusable abstraction rather than introducing a parallel second mechanism. The v0.1.1 scope should stop at canonical label selection plus a shared JSON-facing model for enums and sealed traits; byte-level sum encoding can follow later on top of the same abstraction.
- Neotype-style opaque companion mix-ins: provide a reusable companion trait for opaque wrappers so downstream applications can derive `ByteCodec`, `JsonCodec`, `JsonKeyCodec` (when appropriate), and `Eq` from the representation type without repeating boilerplate in every domain module.
- Tx registry coverage checks: prefer compile-time verification tied to `TxRegistry[Txs]` over open-ended “all sealed subtypes of Tx” checks. This matches the current assembly model better and catches missing handler/reducer evidence when new registered transactions are added.
- Failure metadata foundation: add a lightweight `FailureCode` / `ErrorKey` layer to the core failure model so downstream applications can map failures consistently, without forcing HTTP-specific 4xx/5xx semantics into the library itself.
- Codec laws testkit: package reusable round-trip, deterministic encoding, and ordered-codec law helpers so downstream projects do not keep rewriting the same property suites.
- Derived helper polish for opaque products: improve derivation entry points around `ByteCodec` and related codecs so opaque product wrappers and small domain aggregates require less manual forwarding code.
- Documentation and regression sync discipline: when a public codec rule changes, update examples, fixtures, and property tests in the same milestone so the contract stays coherent.

## Explicitly Out Of Scope For v0.1.1
- Node-level commerce features such as payment escrow flows, debt-backed refund minting, recipient-bound refund liability, and manager release semantics. These are valuable product features, but they do not match the current `sigilaris` release focus on core transaction/model/codec foundations.

## Next Steps
- Validate integration points that still reference legacy TxEnvelope paths.
- Ensure NetworkId usage is documented in public API guides before release freeze.
- Finish the remaining Boolean byte-codec work and update the release note wording so it reflects the actual primitive gaps.
- Lock a canonical JSON enum/discriminator label strategy now, before more public ADTs accumulate compatibility debt, and extract the current JSON sum-type behavior into a reusable label/discriminator strategy instead of introducing a separate competing codec path.
- Prototype a neotype-style companion mix-in that auto-derives byte codecs, JSON codecs, and Eq for opaque wrappers.
- Introduce a small `FailureCode` metadata layer on top of `SigilarisFailure`, while keeping transport-specific error mapping outside core.
- Add reusable codec-law test helpers for round-trip, deterministic encoding, and ordered-codec guarantees.
- Treat byte-level sum encoding as a follow-up phase after the shared label/discriminator model settles; if pursued, expose it as a scodec-inspired `DiscriminatorCodec` rather than a JSON-only special case.
- Introduce a derived helper on `ByteCodec` so opaque products can summon Mirror-based instances without boilerplate.
- Explore representing schema definitions as case classes of `(K_i, V_i)` pairs to simplify tuple conversion.
- Add a `TxRegistry`-oriented compile-time guard so every registered transaction ships with reducer-coverage evidence.

```scala
import scala.compiletime.{erasedValue, summonInline}

sealed trait Tx
final case class CreateNamedAccount(...) extends Tx
final case class UpdateAccount(...) extends Tx

final class TxRegistry[Txs <: Tuple]

/** Compile-time marker that the reducer stack can handle this Tx case. */
trait ReducerCoverage[T <: Tx]

object TxRegistryCoverage:
	inline def ensureAll[Txs <: Tuple]: Unit =
		inline erasedValue[Txs] match
			case _: EmptyTuple =>
				()
			case _: (h *: t) =>
				summonInline[ReducerCoverage[h & Tx]]
				ensureAll[t]

type AppTxs = CreateNamedAccount *: UpdateAccount *: EmptyTuple

// Fails to compile if any registered Tx lacks reducer-coverage evidence
TxRegistryCoverage.ensureAll[AppTxs]
```
